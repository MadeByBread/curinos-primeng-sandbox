Curinos Tokens

Layer 1 is the brand foundation — --curinos-* custom properties generated from Figma collection exports. For how these tokens connect to PrimeNG 7 components, see the Transition page (Layer 2 bridge).

Figma export format

The generator expects the rich export format from Figma variable collections — not the flat token list. Each source file is one collection with variables[], mode definitions, and resolved alias chains.

  • Top-level: name, modes, variables[]
  • Per variable: name, type, resolvedValuesByMode with resolvedValue and optional aliasName

Source file mapping

One Figma collection export maps 1:1 to one generated SCSS partial under src/styles/tokens/sources/.

Source file Figma collection Generated partial CSS prefix Count
curinos-colors.json Curinos Colors _color.scss --curinos-color-* 396
curinos-dimensions.json Curinos Dimensions _dimensions.scss --curinos-dimensions-* 13
curinos-effects.json Curinos Effects _effects.scss --curinos-effects-* 1
curinos-typography.json Curinos Typography _typography.scss --curinos-typography-* 57

Figma "Prime *" collections (Prime Primitive, Prime Component Common, …) are intentionally not exported here — they target a newer PrimeNG token shape. See the PrimeNG bridge (Layer 2) on the Transition page.

Generator rules

scripts/figma-tokens-to-scss.js is collection-driven. Re-run via npm run tokens:build.

  • Collection-driven — one JSON file → one SCSS partial
  • Light mode only — colors ship Light + Dark; only Light is emitted
  • Alias inheritance — same-collection aliases become var(--curinos-…-<alias>, <literal>)
  • Naming — full Figma path slugged to kebab-case (semantic/surface/50 → --curinos-color-semantic-surface-50)
  • Type formatting — colors → hex; floats → px (except unitless typography); font families → quoted stack
